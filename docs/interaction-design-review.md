# Xiyue 交互设计深度评审与竞品对比

**评审日期**：2026-04-11  
**项目**：习乐（Xiyue）音乐学习工具  
**评审范围**：交互方式、用户流程、触觉反馈、手势控制、实时响应、练习体验  
**竞品对比**：Yousician、Simply Piano、TonalEnergy、Soundtrap

---

## 一、竞品交互模式分析

### Yousician 核心交互特征

1. **实时音高检测**：麦克风监听 + 彩色反馈条（绿色=正确，红色=错误）
2. **滚动乐谱**：水平滚动 + 当前音符高亮 + 倒计时预备拍
3. **难度自适应**：根据正确率动态调整速度/简化指法
4. **手势控制**：双指缩放乐谱、长按调出工具菜单
5. **练习模式**：慢速练习（0.5x-1x）、节拍器同步、循环段落
6. **触觉反馈**：错误音符时短震 + 视觉闪烁
7. **成就系统**：完成练习后全屏动画 + 星级评分
8. **语音提示**：每小节结束播报"Good"、"Almost"、"Try again"
9. **进度可视化**：每日练习曲线图、技能树解锁
10. **社交元素**：每周排行榜、好友挑战

### Simply Piano 核心交互特征

1. **等待模式**：自动识别用户弹奏，等待正确音符才继续
2. **分步教学**：左手/右手分离练习，单音→和弦→旋律递进
3. **乐谱简化**：用字母/数字标注指法，降低认知负担
4. **实时评分**：每个音符结束后立即显示 ✓ / ✗，累计正确数
5. **慢速回放**：双击乐谱区域回放当前小节
6. **手势快进**：向右滑动跳过提示，向左滑动重复
7. **练习报告**：练习时长、正确率、卡点小节统计
8. **音量平衡**：伴奏音量/教学语音/用户输入分别调节
9. **模式切换**：学习模式（有提示）→ 演奏模式（无提示）
10. **智能纠错**：识别常见错误（如C大调按成Cm），给出针对性建议

### TonalEnergy 核心交互特征

1. **模拟指针调音器**：实时指针摆动 + 颜色渐变（红色→绿色）
2. **内置节拍器**：视觉闪烁 + 可调节重音拍 + 多种音色（木鱼/牛铃/电子）
3. **波形显示**：实时显示输入音频波形，帮助检查起音/尾音
4. **移调工具**：整体升降调，一键切换乐器（Bb/Eb等）
5. **练习计时器**：设定目标时长，倒计时 + 完成提醒
6. **录音回放**：一键录音 + 波形标记 + 逐帧回放
7. **热键控制**：空格（播放/暂停）、方向键（切换段落）、R（录音）
8. **泛音检测**：显示当前音的泛音列，辅助调整口型/指法
9. **历史曲线**：记录每次调音结果，展示音准稳定性趋势
10. **黑暗模式**：自动跟随系统，护眼模式

### Soundtrap 核心交互特征

1. **网格时间线**：拖拽音频块调整位置/长度，吸附到网格
2. **多轨录音**：叠加录音、编辑音量包络线
3. **节拍器引导**：录制前4拍预备，录制中视觉节拍闪烁
4. **量化功能**：自动对齐音符到最近节拍网格
5. **撤销/重做栈**：无限次撤销，支持历史跳转
6. **快捷键系统**：40+ 键盘快捷键（播放/停止/分割/复制/粘贴）
7. **缩放导航**：Ctrl+滚轮缩放时间线，Alt+拖拽平移
8. **循环选区**：拖拽时间标尺设置循环范围，自动重复播放
9. **MIDI编辑**：钢琴卷帘 + 力度条 + 量化强度滑动条
10. **协作光标**：多人同时编辑时显示其他用户鼠标位置

## 二、三位设计师深度交互评议

### 设计师1：移动交互与手势控制专家

#### 当前优势

1. 滑动根音选择器（SwipeableRootNoteSelector）创新
2. 播放中热切换音阶/根音响应快
3. 通知栏控制符合 Android 规范

#### 问题诊断（12项）

**Q1：缺少手势快捷操作**

- 现状：所有操作需点击按钮
- 用户期望：向右滑动 = 下一练习，向左滑动 = 重复当前，向上滑动 = 调出高级设置
- 改进：实现滑动手势库（SwipeDetector），绑定常见操作

**Q2：BPM滑块无触觉伴随**

- 现状：滑动时静音，仅数值变化
- 竞品：TonalEnergy 滑动节拍器滑块时有轻微震动 + 节拍器滴答声预览
- 改进：`slider.setOnTouchListener` + `VibrationEffect.createOneShot(50, 64)`

**Q3：根音选择器滑动反馈模糊**

- 现状：滑动后无明确选中反馈
- 用户期望：滑动到新根音时震动 + 播放该根音参考音高
- 改进：选中时播放 440Hz 参考音 + 触发 `performHapticFeedback(HapticFeedbackConstants.CONFIRM)`

**Q4：缺少长按上下文菜单**

- 现状：长按音阶卡片无反应
- 竞品：Yousician 长按打开"添加收藏/分享/练习详情"
- 改进：实现 `OnLongClickListener`，弹出 `PopupMenu`

**Q5：无两指缩放乐谱**

- 现状：键盘固定大小
- 用户期望：缩放查看音符细节
- 改进：使用 `ScaleGestureDetector` 包装键盘视图

**Q6：键盘无触摸高亮**

- 现状：仅播放时高亮
- 用户期望：触摸琴键时高亮 + 播放对应音高
- 改进：添加 `OnTouchListener`，按下时临时高亮 + 触发 AudioTrack

**Q7：缺少边缘滑动手势**

- 现状：从屏幕边缘滑动无反应
- 竞品：Simply Piano 从左边缘滑动返回上级
- 改进：使用 `EdgeEffect` + `OnSwipeListener`

**Q8：无双击重置功能**

- 现状：调节参数后难以快速恢复默认
- 用户期望：双击 BPM 滑块恢复 120，双击根音恢复 C
- 改进：`OnDoubleTapListener` 重置到默认值

**Q9：缺少摇杆/触摸板控制**

- 现状：播放/停止需点击小按钮
- 用户期望：大区域触控板（类似音乐播放器）
- 改进：设计浮动播放球（类似 Facebook Messenger），拖拽到屏幕任意位置

**Q10：无压力感应支持**

- 现状：忽略 3D Touch
- 改进：使用 `MotionEvent.getPressure()` 控制音量/力度

**Q11：缺少无障碍手势**

- 现状：TalkBack 无自定义手势
- 改进：添加手势映射：双指上滑 = 切换播放模式，双指下滑 = 切换音阶

**Q12：无导航栏手势适配**

- 现状：底部手势导航冲突
- 改进：`WindowInsetsController` 检测手势区域，避免交互冲突

### 设计师2：实时反馈与音频交互专家

#### 当前优势

1. 实时键盘预览同步良好
2. 后台播放 + 通知栏控制完善
3. 27 种动画丰富

#### 问题诊断（12项）

**Q1：无实时节拍器反馈**

- 现状：播放时无视觉节拍引导
- 竞品：TonalEnergy 播放时屏幕边缘周期性闪烁（红→绿→红）
- 改进：添加全屏半透明遮罩，按 BPM 节奏闪烁

**Q2：无音频输入检测**

- 现状：纯输出应用，无法判断用户弹奏
- 竞品：Yousician 通过麦克风检测用户弹奏音符
- 改进：集成 `AudioRecord` + FFT 音高检测，显示"匹配度"进度条

**Q3：错误容忍无渐进难度**

- 现状：播放模式固定，无自适应
- 用户期望：连续正确3次后加速，连续错误3次后减速
- 改进：实现自适应算法：`correctStreak > 3 ? BPM += 5 : BPM -= 5`

**Q4：缺少练习模式细分**

- 现状：仅上行/下行/上下行/和弦齐奏/琶音
- 竞品：Simply Piano 还有：仅右手/仅左手/分手慢练/合手
- 改进：添加"慢速练习"（0.5x）、"跟奏模式"（等用户弹完再播下一个）

**Q5：无循环段落功能**

- 现状：播放整个音阶，无法聚焦难点
- 用户期望：长按音符设置循环起点/终点
- 改进：添加`LoopRange`选择器，拖拽时间轴设置循环

**Q6：缺少预备拍**

- 现状：点击播放立即开始
- 用户期望：4拍倒计时，让用户准备
- 改进：播放前 `CountDownTimer`（4,3,2,1,Go）

**Q7：无录音回放功能**

- 现状：仅播放预设音色
- 竞品：Soundtrap 录制用户演奏并回放对比
- 改进：集成 `MediaRecorder`，保存到 `.wav`，提供对比播放

**Q8：无视觉音高曲线**

- 现状：仅有键盘高亮
- 竞品：TonalEnergy 显示实时音高波形 + 目标音高线
- 改进：添加 `WaveformView`，绘制当前播放音的频谱

**Q9：缺少音量包络控制**

- 现状：固定力度
- 用户期望：自定义每个音的力度（强/弱/渐强）
- 改进：添加力度滑块，用 `AudioTrack.setVolume` 动态调整

**Q10：无节拍器子分拍**

- 现状：仅 BPM 控制
- 竞品：TonalEnergy 支持 16 分音符子拍（如 4/4 拍下每拍再分 4 个提示音）
- 改进：添加"子拍精度"选项（无/2分/4分/8分/16分）

**Q11：无乐谱跟随高亮**

- 现状：仅键盘高亮，无乐谱视图
- 用户期望：滚动乐谱 + 当前小节高亮
- 改进：集成 `MusicXML` 解析 + 自定义 `ScoreView`

**Q12：缺少练习报告**

- 现状：无练习数据记录
- 竞品：Simply Piano 显示练习时长、正确率、进步曲线
- 改进：使用 `Room` 存储练习记录，生成统计图表

### 设计师3：学习路径与智能引导专家

#### 当前优势

1. 音阶/和弦种类丰富（13+12）
2. 搜索筛选功能完整
3. 实时切换体验流畅

#### 问题诊断（12项）

**Q1：无新手引导流程**

- 现状：打开 App 直接进入主界面
- 用户期望：首次打开引导选择乐器（钢琴/吉他/其他）
- 改进：添加 `OnboardingActivity`，设置用户水平（初级/中级/高级）

**Q2：无学习路径推荐**

- 现状：所有音阶平铺展示
- 竞品：Yousician 按难度排序（C大调 → G大调 → D大调...）
- 改进：添加"推荐学习顺序"侧边栏，标注难度星级

**Q3：缺少练习提醒系统**

- 现状：用户需主动打开
- 用户期望：每天固定时间提醒练习
- 改进：集成 `WorkManager` + 通知渠道

**Q4：无智能错误诊断**

- 现状：无错误分析
- 竞品：Simply Piano 检测常见错误（如"你弹的是F#，但需要F"）
- 改进：记录用户常错音符，生成"薄弱环节"列表

**Q5：缺少成就徽章系统**

- 现状：无激励体系
- 用户期望：完成里程碑获得徽章（如"10小时练习"、"掌握5个音阶"）
- 改进：定义成就规则 + `BadgeManager` 触发通知

**Q6：无每日挑战**

- 现状：静态内容
- 竞品：Yousician 每日任务（如"练习音阶10分钟"）
- 改进：服务器下发每日挑战 + 本地缓存

**Q7：缺少难度分级**

- 现状：13种音阶无难度标签
- 用户期望：初级（C,G,D,F）、中级（A,E,Bb）、高级（F#等）
- 改进：添加 `difficulty` 字段，提供筛选器

**Q8：无练习笔记功能**

- 现状：无法记录心得
- 用户期望：在每个音阶下添加个人笔记
- 改进：使用 `SharedPreferences` 存储用户笔记

**Q9：缺少社交分享**

- 现状：无分享功能
- 竞品：Yousician 支持分享成就到微信/朋友圈
- 改进：集成系统分享，生成练习卡片截图

**Q10：无云端同步**

- 现状：数据仅本地
- 用户期望：多设备同步进度
- 改进：接入 Firebase/AWS，同步用户数据

**Q11：缺少音阶对比功能**

- 现状：一次只能选一个
- 用户期望：并排对比两个音阶的指法/音高
- 改进：添加"对比模式"，分屏显示两个键盘

**Q12：无离线练习模式**

- 现状：已支持离线播放
- 改进：预下载所有音阶的 PCM 数据，确保无网络可用

## 五、验收标准

1. **手势交互**：滑动根音选择器时震动 + 播放参考音
2. **触觉反馈**：所有滑动/点击操作有适当震动反馈
3. **预备拍**：播放前有 4 拍倒计时提示音和视觉
4. **节拍器视觉**：播放时屏幕边缘按 BPM 节奏闪烁
5. **长按菜单**：长按音阶卡片弹出"收藏/分享/详情"
6. **录音功能**：能录制 30 秒演奏并回放
7. **数据统计**：显示今日练习时长、本周趋势图
8. **新手引导**：首次启动有 3 步引导页
9. **难度筛选**：可按初级/中级/高级过滤音阶列表
10. **双击重置**：双击 BPM 滑块恢复 120

## 六、风险与注意事项

1. **震动权限**：Android 需在 manifest 声明 `<uses-permission android:name="android.permission.VIBRATE" />`
2. **录音权限**：需动态请求 `RECORD_AUDIO` 权限
3. **存储权限**：保存录音文件需 `WRITE_EXTERNAL_STORAGE`（Android 10+ 使用 MediaStore）
4. **性能**：波形绘制使用 `SurfaceView` 避免卡顿
5. **兼容性**：两指缩放需 Android 3.0+（全支持）
6. **电池消耗**：实时麦克风检测会消耗电量，默认关闭
