## 三、并行执行要点

### 1. 代码隔离策略

- **组1（手势）**：修改 `ui/components/` 下的触摸交互类，不影响音频流
- **组2（音频）**：主要修改 `playback/` 和 `audio/`，注意线程安全
- **组3（学习）**：新增 `learning/` 模块，独立 Room 数据库

### 2. 每日同步（15 分钟）

- 早晨站会：确认进度、接口变更、阻塞问题
- 晚上 PR：各自提交 PR，由 D 工程师集成测试

### 3. 共用资源管理

- **震动权限**：A、B 组共用，在 `AndroidManifest.xml` 统一声明
- **录音权限**：B 组单独处理，动态请求
- **数据库**：C 组独享 `AppDatabase`，其他组只读

## 四、关键验收点（每个 Sprint 结束）

### Sprint 1 验收

- [ ] 滑动根音选择器：震动 + 播放参考音
- [ ] BPM 滑块：滑动震动，双击恢复 120
- [ ] 长按卡片：弹出菜单（收藏/分享/详情）
- [ ] 两指缩放键盘：缩放比例 0.5x - 2x
- [ ] 播放前：4 拍倒计时（视觉 + 滴答声）
- [ ] 播放时：屏幕边缘按 BPM 闪烁
- [ ] 录音：能录制 30 秒并回放
- [ ] 首次启动：3 步引导页
- [ ] 音阶列表：可按难度筛选

### Sprint 2 验收

- [ ] 边缘滑动：左边缘返回，右边缘前进
- [ ] 触摸键盘：按下高亮 + 播放对应音
- [ ] 浮动播放球：可拖拽，点击播放/停止
- [ ] 波形视图：实时显示频谱
- [ ] 力度控制：每个音可独立调节音量
- [ ] 节拍器子分拍：支持 16 分音符提示
- [ ] 数据统计：显示今日练习时长 + 周趋势图
- [ ] 自适应速度：连续正确 3 次 +5 BPM，错误 3 次 -5 BPM
- [ ] 成就：完成 5 个音阶解锁"新手"徽章

### Sprint 3 验收

- [ ] 3D Touch：重按控制音量
- [ ] 无障碍：双指上下滑切换播放模式
- [ ] 手势适配：底部导航栏不冲突
- [ ] 滚动乐谱：播放时当前小节居中高亮
- [ ] 循环段落：拖拽时间轴设置 A-B 循环
- [ ] 分享：生成练习卡片截图并分享
- [ ] 云端：登录后同步进度（至少支持 Google 登录）
- [ ] 每日挑战：每天更新 3 个任务

## 五、风险缓解

| 风险               | 概率 | 影响 | 缓解措施                              |
| ------------------ | ---- | ---- | ------------------------------------- |
| 手势与系统导航冲突 | 中   | 高   | 提前在 Pixel/OnePlus 测试边缘手势     |
| 录音导致音频延迟   | 高   | 中   | 使用低延迟 `AudioRecord` + 后台线程   |
| 数据库升级崩溃     | 低   | 高   | C 组编写迁移测试                      |
| 3D Touch 设备兼容  | 低   | 中   | 运行时检测 `isTouchPressureSupported` |
| 云端同步冲突       | 中   | 高   | 采用 Last-Write-Wins 策略 + 冲突提示  |

## 六、工具与环境

- **版本控制**：功能分支 `feature/sprintX-组名`，合并到 `develop`
- **自动化测试**：JUnit + Espresso（覆盖核心手势）
- **性能监控**：LeakCanary + 自定义帧率检测（滑动时 > 60fps）
- **云服务**：Firebase Auth + Firestore（Sprint 3 集成）
